# use-ZAP.py

Gọi ZAP pentest các URL đã lọc và lấy kết quả raw:

```bash
py tools/use-ZAP.py -i <endpoint_file> -b <base_url> --sid <PHPSESSID> --fast-scan -o <output_file> --proxy-host <proxy_host> --proxy-port <proxy_port>

# Example:
py tools/use-ZAP.py -i katana.filtered.txt -b http://192.168.153.200/DVWA/ --sid "1lsg7f8o7fjg75agf20t0dp7gi" --fast-scan -o zap_results.json --proxy-host 192.168.153.130 --proxy-port 8080
```

# agent-scaner.py

### ROLE: Targeted Probing & Verification (Phase 3)

--- STRICT RULES ---

- LIMIT: Max 03 'bulk-send' calls total.
- PAYLOADS: Max 03 payloads per endpoint per call.
- ACCESS: Agent MUST NOT read raw response files. Snippets only via 'search'.

--- COMMAND: bulk-send ---

- Description: Multi-threaded request execution; saves body to disk.
- Usage: py tools/super_scanner.py bulk-send -m <manifest.json> -s <session_file>
- Input (manifest.json):

```json
[
	{
		"url": "http://192.168.153.200/path/",
		"method": "POST",
		"param": "id",
		"values": ["p1", "p2", "p3"]
	}
]
```

- Output (Metadata):

```json
[{ "endpoint": "...", "response_id": "a1b2c3d4", "status_code": 200, "ms": 45, "len": 1024 }]
```

--- COMMAND: search ---

- Description: Keywords lookup in saved responses; returns snippets; auto-deletes files.
- Usage: py tools/super_scanner.py search --ids <id1 id2> -k <kw1 kw2>
- Output (Snippets):

```json
{
	"a1b2c3d4": {
		"hits": [{ "kw": "sql syntax", "snip": "...error in your SQL syntax near..." }]
	}
}
```

--- AGENT WORKFLOW ---

1. Identify high-risk nodes from 'zap_reduced.json'.
2. Craft 'manifest.json' (Max 3 payloads/target).
3. Execute 'bulk-send' -> Receive Response IDs.
4. Execute 'search' using error strings or payload tags.
5. Confirm vulnerability via snippets, status_code, or length/time diffs.
